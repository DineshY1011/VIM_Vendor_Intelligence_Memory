Memory schema reference
Baseline fields — {vendor_id}:baseline
vendor_name
Full legal vendor name from contract
e.g. "Twilio Inc."
agreed_price
Exact pricing at signing with units
e.g. "$0.0075 per SMS"
sla_terms
Verbatim SLA quote from contract
e.g. "99.95% uptime guaranteed"
contract_start
Contract effective date
YYYY-MM-DD
renewal_date
Next renewal trigger date
YYYY-MM-DD
rep_name
Vendor account rep who signed
e.g. "Jordan Mills"
Event fields — {vendor_id}:events
date
Date of the source document
YYYY-MM-DD
fact_type
One of five valid categories
price_change · sla_violation · term_change · commitment · incident
value
Specific value or statement extracted
e.g. "$0.0081 per SMS"
summary
One-sentence LLM-generated description
e.g. "Price raised 8% with no advance notice"
severity
Set by drift classifier — 1 minor · 2 notable · 3 breach
Integer 1, 2, or 3
source_doc
Source filename — cited by agent in brief
e.g. "invoice_2023_01.pdf"
Vendor configuration
lowercase · no spaces · e.g. twilio, aws, salesforce
{vendor_id}:baseline or {vendor_id}:events
Document IDs auto-numbered per upload batch
Ingest mode
Baseline write_baseline()
Original signed contract. Writes {vendor_id}:baseline once. Never overwritten — frozen as drift reference point.
Normal event append_event()
Post-contract documents — invoices, emails, call transcripts, renewals. Appends to {vendor_id}:events append-only list.
